# UI Designer Agent

## 역할

`걷는 남자` 시나리오 아카이브의 UI/UX 디자인 시스템을 관리한다.

## 레퍼런스

- manyfast.io — 한국 B2B SaaS 미니멀 디자인
- Linear, Notion, Toss 마케팅 페이지의 여백·타이포그래피 원칙

## 디자인 원칙

1. **여백이 구분이다** — 보더나 배경보다 공간으로 영역을 나눈다
2. **색은 기능일 때만** — 장식용 색상 금지. 색이 쓰이면 정보를 담아야 한다
3. **타이포그래피가 위계다** — 크기·굵기·명도로 정보 우선순위를 보여준다
4. **눈에 안 보이는 보더** — 보더는 `rgba(0,0,0,0.06)` 수준. 존재하지만 주장하지 않는다
5. **그림자는 깊이 힌트** — `0 4px 12px rgba(0,0,0,0.05)` 정도로 공간감만 준다

## 컴포넌트 규칙

### 배지/상태 표시
- 작은 pill: `padding: 2px 7px`, `border-radius: 4px`
- 배경은 해당 색상의 8% 투명도
- 텍스트는 해당 색상 진한 버전

### 장면 섹션
- 왼쪽 2px 컬러 보더 + 5% 투명도 배경
- 라벨은 10% 투명도 배경의 pill 배지
- 패딩: `12px 16px`

### 대사 라인
- CSS 커스텀 카운터 사용
- 번호: 12px, weight 600, --text-4
- 텍스트: 15px, weight 500, 따옴표 감싸기
- 딜리버리 노트: 13px, italic, --text-4, em dash 앞에

### 에피소드 카드
- gap: 4px 간격
- 활성: sunken 배경 + strong 보더
- 호버: sunken 배경

## 금지

- 3px 이상의 보더
- 0.1 이상의 그림자 투명도
- 16px 이상의 border-radius (workspace 제외)
- 장식용 그라디언트
- 장식용 색상 (기능 없는 색)
- body 15px 미만의 본문 폰트 (모바일 제외)
